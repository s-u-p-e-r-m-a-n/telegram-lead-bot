# 🤖 Telegram Lead Bot

Telegram-бот для приема заявок и быстрой связи с клиентом.

Подходит для ситуаций, когда нужно удобно принимать заявки в Telegram, сохранять их в базе и сразу получать уведомление о новой заявке.

Проект опубликован как demo/showcase Telegram-бота для бизнеса.

## ✨ Что умеет бот

- помогает клиенту оставить заявку по шагам
- предлагает выбрать категорию заявки
- запрашивает имя, телефон и комментарий
- сохраняет заявку в PostgreSQL
- отправляет администратору оформленную заявку в Telegram
- показывает последние заявки через `/last_leads`
- выводит заявки от новых к более ранним
- показывает demo-результат — как заявка выглядит для администратора

## 🛠 Технологии

- Java 17
- Spring Boot
- Spring Data JPA
- PostgreSQL
- Flyway
- Docker Compose
- JUnit 5
- Mockito

## 📌 Команды

- `/start` — начать работу
- `/help` — помощь
- `/last_leads` — последние заявки для администратора
## ✅ Тесты

В проекте добавлены минимальные unit-тесты на ключевую бизнес-логику:

- сохранение заявки (`LeadServiceTest`)
- сбор статистики (`StatsServiceTest`)
- шаги пользовательской сессии (`UserSessionServiceTest`)

## ⚙️ Переменные окружения

В проекте есть файл `.env.example`.

Перед запуском создай на его основе свой файл `.env` в корне проекта и укажи свои значения:

```env
BOT_TOKEN=your_bot_token_here
ADMIN_CHAT_ID=123456789
DB_URL=jdbc:postgresql://postgres:5432/telegram_lead_bot
DB_USERNAME=postgres
DB_PASSWORD=postgres
STATS_CRON=0 30 7 * * *
```
Что нужно подставить:
- `BOT_TOKEN` — токен вашего Telegram-бота
- `ADMIN_CHAT_ID` — chatId администратора, который будет получать уведомления
- `DB_USERNAME` и `DB_PASSWORD` — данные для подключения к PostgreSQL

## 🚀 Запуск через Docker Compose

### 1. Подготовь `.env`

Создай файл `.env` в корне проекта на основе `.env.example` и укажи свои значения:
- токен Telegram-бота
- chatId администратора
- параметры подключения к базе

### 2. Запусти проект

```bash
docker compose up -d --build
```
### 3. Проверь работу бота
  - открой Telegram
  - найди своего бота
  - отправь команду /start
  - пройди сценарий создания заявки
### 4. Остановить проект
```bash
   docker compose down
```
